p2c_1d

Synopsis

Reads a field configuration in one spatial dimension from a legacy ASCII VTK file with scalars in polar field coordinates, and writes it to a new VTK file, converting the scalars to Cartesian coordinates.

Usage

p2c_1d [arguments]

Arguments

All arguments are entered as a name (hyphen-minus followed by a two-character combination) separated from the associated value by whitespace. An argument without a value is not allowed. If an argument is not provided on the command line, the default value shown below [between square brackets] is used:

-if ["initial"]
Name of input file. Do not use file extensions; ".vtk" is automatically appended.
-of ["p2ced"]
Name of output file. Do not use file extensions; ".vtk" is automatically appended.

Details

Input (-if) and output filenames (-of) should require no explanation.

All momenta are set to 0.

Examples

p2c_1d
Run p2c_1d using the defaults for all argument values.
p2c_1d -if polar_file -of cartesian_file
Read "polar_file.vtk", write "cartesian_file.vtk".

See also

The output from p2c_1d is usually input for evolve_1d.

3D counterpart: p2c.
